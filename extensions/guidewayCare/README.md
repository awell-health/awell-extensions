# Guideway Care

Guideway Care provides an AI-powered observation summarization engine that processes raw patient interaction text — phone call transcripts, chat logs, or clinical notes — and returns structured clinical observations, a narrative summary, and a follow-up action list.

## Extension settings

| Setting | Required | Description |
|---------|----------|-------------|
| **API Key** | Yes | Your Guideway Care API key (`X-API-Key` header). Contact Guideway Care to obtain one. |
| **Base URL** | No | Override the Guideway Care API base URL. Defaults to the production endpoint. Useful for pointing at a staging environment. |

## Actions

### Summarize Observation

Submit a patient call transcript or interaction text and receive AI-generated clinical observations, a summary, and a follow-up list.

**Inputs**

| Field | Required | Description |
|-------|----------|-------------|
| **Source Text** | Yes | The full patient call transcript or interaction text to analyze. |
| **Care Flow ID** | No | Identifier for the Awell care flow or pathway associated with this interaction. |
| **Processed Datetime** | No | ISO 8601 datetime for when the interaction occurred. Defaults to the current time if omitted. |
| **Source Type** | No | The type of interaction source, e.g. `phone_call`, `chat`, or `note`. |
| **Source ID** | No | A unique identifier for this source record. Auto-generated by Guideway Care if omitted. |
| **Context** | No | A JSON object of key-value string pairs injected as known context into the AI prompt, allowing you to tailor the analysis (e.g. patient demographics, program enrollment). |

**Outputs**

| Data Point | Type | Description |
|------------|------|-------------|
| `summary` | string | A brief plain-text overall summary of the interaction. |
| `observationsSummaryFormatted` | string | HTML-formatted rich text covering all extracted observation topics — ready to render in a clinical UI. |
| `followupFormatted` | string | HTML-formatted list of items that require follow-up action. |
| `observations` | json | JSON array of structured observation objects. Each observation includes `name`, `display_name`, `domain`, `value`, `detail`, `evidence`, and `confidence` (`high` / `medium` / `low`). |
| `sourceId` | string | The source ID returned by the API (either the one you provided or the auto-generated value). |
| `processedAt` | string | ISO 8601 timestamp of when processing occurred. |
| `processingTimeMs` | number | Total processing time in milliseconds. |
| `promptVersion` | number | Version of the AI prompt used — useful for audit trails when prompts are updated. |
| `totalTokens` | number | Total LLM tokens consumed by this request. |
| `estimatedCost` | number | Estimated cost of the LLM call. |
